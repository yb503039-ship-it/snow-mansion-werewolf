# 吹雪の屋敷 人狼 / Snow Mansion Werewolf

身内で遊ぶための、ネット公開できる人狼Webアプリの試作版です。
各プレイヤーは自分のスマホで部屋URLに入り、自分の役職確認・夜行動・投票を行えます。

## 現在できること

- 部屋作成・部屋コード参加
- 参加用URL共有
- 4〜20人対応
- GMありモード：ホストがフェーズを進める
- GMなしモード：タイマーと行動完了で自動進行
- 初級役職のみ
  - 市民
  - 人狼
  - 占い師
  - 狩人
  - 霊能者
  - 狂人
- おすすめ配役の自動設定
- 役職ランダム配布
- 各プレイヤーの個別役職画面
- 人狼の仲間表示
- 夜行動
  - 人狼：襲撃
  - 占い師：占い
  - 狩人：護衛
- 投票・決選投票・同数時ランダム処刑
- 処刑見送り
- 勝利判定
  - 市民陣営：人狼が全滅
  - 人狼陣営：人狼数が市民数以上
- 全体チャット・人狼チャット・墓場チャット

## 必要なもの

- Node.js 20以上
- npm

## ローカルで動かす

```bash
npm install
npm start
```

起動後、ブラウザで開きます。

```text
http://localhost:3000
```

同じPC内で試す場合は、複数タブで開いて動作確認できます。

## ネット公開する場合

Render / Railway などの Node.js Web Service にアップロードすれば動きます。

### Renderの例

- Build Command

```bash
npm install
```

- Start Command

```bash
npm start
```

- Environment

```text
NODE_VERSION=20
```

### Railwayの例

RailwayにGitHubリポジトリを連携し、Node.jsアプリとしてデプロイします。
通常は `npm start` が自動で使われます。

## 遊び方

1. ホストが部屋を作成
2. 参加用URLをLINEなどで共有
3. プレイヤーが名前を入力して参加
4. ホストが配役・時間設定を確認
5. ゲーム開始
6. 各プレイヤーはスマホで自分の役職を確認
7. 夜行動 → 昼の話し合い → 投票 → 処刑 → 夜、を繰り返す

## 注意

この版は試作版です。
ゲーム情報はサーバーのメモリ上に保存されます。
サーバーが再起動すると部屋情報は消えます。

本格運用する場合は、次の改善が必要です。

- 再接続処理の強化
- 部屋パスワード
- QRコード生成
- 観戦者モード
- 役職希望
- 役職欠け
- 中級・上級役職
- データベース保存
- 不正操作対策
